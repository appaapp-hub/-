# 京都唐辛子 エジプトピクルス風ソース - ランディングページ

京都の唐辛子とエジプトの食文化が融合したホットスパイシーソースの商品紹介・お問い合わせ用ランディングページです。

---

## はじめての方へ（パソコンが苦手でも大丈夫）

この説明では、**自分のパソコンでソースの紹介ページを表示する**までの手順を、できるだけやさしく書いています。  
「ターミナル」や「コマンド」と聞くと難しそうに感じるかもしれませんが、**書いてある文字をそのまま入力する**だけで大丈夫です。ひとつずつ進めてみてください。

**このあと出てくる言葉の意味（ざっくり）：**

| 言葉 | 意味（イメージ） |
|------|------------------|
| Node.js（ノード・ジェイエス） | このページを動かすために必要な、無料のソフト |
| ターミナル | 文字で指示を送るための窓。Cursor の下や横に出てきます |
| コマンド | ターミナルに打ち込む「お願いの文」。ここでは英語のような短い文字です |
|  Enter を押す | コマンドを打ち終えたら、キーボードの「Enter」キーで確定します |

---

## 開き方（6つのステップ）

### ステップ1：Node.js が入っているか確認する

このページを表示するには、**Node.js** という無料のソフトがパソコンに入っている必要があります。

**やること：**

1. キーボードの **Windows マーク** を押す（または画面左下の Windows ボタンをクリック）
2. 「**cmd**」と入力する
3. 出てきた **「コマンド プロンプト」** をクリックして開く
4. 黒い画面が出たら、次の文字を **そのまま** 打ち込む（コピー＆ペーストでもOK）  
   ```text
   node -v
   ```
5. **Enter キー** を押す

**結果の見方：**

- **「v」で始まる数字**（例：v20.10.0）が表示された → Node.js は入っています。**ステップ2** へ進んでください。
- **「'node' は認識されていません」** や **「見つかりません」** と出た → Node.js が入っていません。下の「Node.js を入れる」をやってから、もう一度ステップ1を試すか、そのままステップ2に進んでください。

**Node.js を入れる（入っていない人だけ）：**

1. インターネットで **nodejs.org** を開く（「Node.js 公式」で検索してもOK）
2. 緑色の **「LTS」** と書いてあるほうの **「ダウンロード」** をクリック
3. ダウンロードされたファイルをダブルクリックして開く
4. 画面の指示に従って **「次へ」** などで進め、最後に **「インストール」** を選ぶ
5. 終わったら **パソコンをいったん再起動** するか、**Cursor を閉じて開き直す**

---

### ステップ2：このプロジェクトを Cursor で開く

**やること：**

1. **Cursor** を起動する（デスクトップやスタートメニューのアイコンから）
2. 画面上部の **「ファイル」** をクリック
3. メニューから **「フォルダーを開く」** をクリック
4. 開いたウィンドウで、次の場所に移動する：  
   **このPC** → **C:** → **Users** → **81708** → **プロジェクトA**  
   （または、上のアドレス欄に `c:\Users\81708\プロジェクトA` と入力して Enter）
5. **「プロジェクトA」** のフォルダをクリックして選び、**「フォルダーの選択」** をクリック

**うまくいった目安：** 左側に「プロジェクトA」の中身（フォルダやファイル名）が一覧で出ていればOKです。

---

### ステップ3：ターミナルを開く

**ターミナル** は、下や横に出す「文字を打ち込む小さな窓」です。ここにこれからコマンドを打ちます。

**やること：**

1. Cursor の一番上にあるメニューで **「ターミナル」** をクリック
2. **「新しいターミナル」** をクリック
3. 画面の **下** または **横** に、黒や青っぽい背景のパネルが開く
4. その中に、`PS C:\Users\81708\プロジェクトA>` のような文字が出ていればOK  
   （環境によって少し違う場合もあります。プロジェクト名が出ていれば大丈夫です）

**別の方法：** キーボードで **Ctrl + `**（「Ctrl」キーを押したまま、キーボード左上の「`」キーを押す）でもターミナルを開けます。

---

### ステップ4：必要なものを用意する（初めてこのプロジェクトを開くときだけ）

**やること：**

1. ステップ3で開いた **ターミナル** のところをクリックして、入力できる状態にする
2. 次の文字を **そのまま** 打ち込む（コピー＆ペーストでOK）：  
   ```text
   npm install
   ```
3. **Enter キー** を押す
4. そのまま待つ（数十秒〜2分くらいかかることがあります。たくさん文字が流れても大丈夫です）
5. 流れが止まって、また `PS C:\...>` のような表示に戻れば完了です

**注意：** 2回目以降、このプロジェクトを開くときは **このステップは飛ばして** ステップ5へ進んでください。

---

### ステップ5：ページを表示する準備（サーバーを起動する）

**やること：**

1. 同じ **ターミナル** で、次の文字を **そのまま** 打ち込む：  
   ```text
   npm run dev
   ```
2. **Enter キー** を押す
3. 少しすると、次のような行が表示されます：  
   ```text
   ➜  Local:   http://localhost:5173/
   ```  
   これが「ページの住所」です。

**重要：** このあと、**ターミナルの窓は閉じないでください**。閉じるとページが表示されなくなります。そのままにして、次のステップへ。

---

### ステップ6：ブラウザでページを開く

**やること：**

1. **ブラウザ**（Chrome、Edge、Firefox など）を開く
2. いちばん上の **アドレス欄**（URLを入力するところ）をクリック
3. 次の文字を入力する（コピー＆ペーストでOK）：  
   ```text
   http://localhost:5173
   ```
4. **Enter キー** を押す

**うまくいったら：** 「京の辛さ、エジプトの魂。」という見出しの、ソースの紹介ページが表示されます。これで完了です。

## よくあること・困ったとき

- **ページを閉じたいとき**  
  ターミナルで **Ctrl + C**（Ctrl キーを押したまま C キーを押す）を押すと、サーバーが止まります。その後、ターミナルは閉じて大丈夫です。

- **次にまた開くとき**  
  **ステップ2**（Cursor でフォルダを開く）→ **ステップ3**（ターミナルを開く）→ **ステップ5**（`npm run dev`）→ **ステップ6**（ブラウザで開く）だけでOKです。ステップ1と4はやり直さなくて大丈夫です。

- **「npm は認識されていません」と出る**  
  Node.js が入っていないか、入れ直したあとに Cursor を開き直していない可能性があります。ステップ1の「Node.js を入れる」をやり、**Cursor を一度閉じて開き直して** からもう一度試してください。

- **赤い文字（エラー）がいっぱい出る**  
  まず、打ち込んだ文字が **`npm install`** と **`npm run dev`** で合っているか確認してください。余計なスペースや typo がないかも見直してみてください。

## セットアップ

```bash
npm install
```

## 開発サーバー

```bash
npm run dev
```

ブラウザで http://localhost:5173 を開いてください。

## ビルド

```bash
npm run build
```

`dist` フォルダに静的ファイルが出力されます。Vercel や Netlify にそのままデプロイできます。

## お問い合わせフォーム（Formspree）

フォーム送信先を Formspree で設定する場合：

1. [Formspree](https://formspree.io) でアカウントを作成し、新しいフォームを作成
2. 発行されたフォームのエンドポイント（例: `https://formspree.io/f/xxxxxxxx`）をコピー
3. プロジェクトルートに `.env` を作成し、次を追加：

```
VITE_FORMSPREE_ENDPOINT=https://formspree.io/f/あなたのフォームID
```

4. 開発サーバーを再起動

`.env` を設定しない場合、送信はプレースホルダーURLに向かうため、本番では必ず設定してください。

## 画像の差し替え

- **Hero**: `src/components/Hero.tsx` 内の `img` の `src` を実際のソース・瓶の写真に変更
- **Gallery**: `src/components/Gallery.tsx` の `galleryItems` 配列内の `src` を料理写真のURLに変更

## デプロイ

- **Vercel**: リポジトリを連携し、ビルドコマンド `npm run build`、出力ディレクトリ `dist` でデプロイ
- **Netlify**: 同様にビルド設定後、環境変数 `VITE_FORMSPREE_ENDPOINT` を設定
